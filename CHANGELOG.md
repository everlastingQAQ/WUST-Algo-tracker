# Changelog

本项目使用语义化版本管理：`v主版本.次版本.修订版本`。

- 修订版本：修复 bug、小接口补充、不改变核心功能，例如 `v1.1.2 -> v1.1.3`。
- 次版本：新增一组完整功能或数据能力，例如 `v1.1.2 -> v1.2.0`。
- 主版本：大规模架构调整、数据库模型重构或接口不兼容，例如 `v1.x.x -> v2.0.0`。

## v1.1.5

- 新增用户服务操作日志查询接口 `/v1/user/system/operation-logs`。
- 新增核心数据服务操作日志查询接口 `/v1/core/operation-logs`。
- 新增 `feature_snapshots` 表和 `/v1/core/snapshot` 读写接口，支持训练周报和成就快照。
- 新增全站成就快照接口 `/v1/core/achievement/global-snapshot`，后端聚合成就解锁比例和站内百分位上下文。
- 统计口径说明补充平台明细、重爬队列和大型账号分批写入的差异原因。
- 抓取审计说明细化 Codeforces、洛谷、牛客、QOJ、AtCoder 与 OJ 主页统计不一致的常见原因。
- CodeForces 30 分钟冷却保护下沉到绑定、重试和队列执行层，避免通过其他入口频繁触发 CF 抓取。

## v1.1.3

- 后台角色调整不再允许直接授予管理员角色。
- 禁止修改管理员账号角色，降低后台误操作和越权风险。
- 禁止管理员修改自己的角色，避免将最后一个管理员降级。
- 本人修改密码必须校验旧密码，包括管理员本人。
- 管理员只能重置非管理员账号密码，不能重置其他管理员密码。
- 群发消息权限改为查询数据库角色，避免旧 JWT 角色导致的权限残留。
- README 补充 v1.1.3 工程质量与权限补洞说明。
